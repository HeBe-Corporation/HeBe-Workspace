# AI 오케스트레이션 커맨드 가이드

> Claude Code를 **지휘자**처럼 사용하는 3가지 핵심 커맨드

---

## 왜 커맨드가 필요한가?

Claude에게 그냥 질문하면 매번 다른 방식으로 답합니다.
커맨드는 **Claude의 동작 모드를 지정**합니다.

| 방식 | 결과 |
|------|------|
| 그냥 질문 | 매번 다른 형식, 일관성 없음 |
| 커맨드 사용 | 구조화된 출력, 예측 가능한 동작 |

---

## 3가지 핵심 커맨드

### 1. `/gather` - 정보 수집 모드

**용도**: 작업 시작 전 필요한 정보를 체계적으로 수집

**동작**:
- 5-7개의 구조화된 질문 생성
- 핵심 요구사항, 제약조건, 성공 기준 파악
- 맥락과 배경 정보 수집

**사용 예시**:
```
/gather 새 프로젝트 기획
/gather 고객 요구사항 정리
/gather 문제 상황 파악
```

**출력 형식**:
```
**Information Gathering**
1. [핵심 요구사항 질문]
2. [제약/맥락 질문]
3. [성공 기준 질문]
4. [세부사항 질문]
5. [예외 상황 질문]
```

---

### 2. `/reframe` - 이해 확인 모드

**용도**: 대화 내용을 정리하고 상호 이해 확인

**동작**:
- 지금까지의 대화를 구조화하여 요약
- 목표, 요구사항, 제약조건, 가정 정리
- 사용자 확인 후 다음 단계 진행

**사용 예시**:
```
/reframe
(긴 대화 후) /reframe 지금까지 내용 정리해줘
```

**출력 형식**:
```
## My Understanding

[상황/맥락 요약 2-3문장]

**Core Objective:** [달성해야 할 것]

**Key Requirements:**
- [요구사항 1]
- [요구사항 2]

**Constraints:**
- [제약조건]

**Success:** [원하는 최종 상태]

**Assumptions:**
- [가정 1]
- [가정 2]

맞나요? [다음 행동]으로 진행할까요?
```

---

### 3. `/truth` - 사실 기반 분석 모드

**용도**: 정확하고 객관적인 분석이 필요할 때

**동작**:
- 증거 기반 분석
- 사실/추론/추측 구분
- 신뢰도 점수 제공
- 불확실성 투명하게 표시

**사용 예시**:
```
/truth 이 사업 아이디어의 현실성
/truth 경쟁사 A vs B 비교
/truth 이 결정의 리스크
```

**출력 형식**:
```
[직접적인 답변 - 150단어 이하]

**Facts:** [검증 가능한 사실]

**Inferences:** [논리적 추론]

**Speculation:** [명확히 표시된 추측]

**Confidence:** X% [근거]

**Trade-offs:**
- Pro: [장점]
- Con: [단점]

**Data Gaps:** [부족한 정보]
```

## 실전 워크플로우

### 예시: 새 서비스 기획

```
1. /gather 새 서비스 아이디어
   → 5-7개 질문으로 요구사항 수집

2. (질문에 답변 후)
   /reframe
   → 이해한 내용 확인

3. /truth 이 서비스의 시장성
   → 객관적 분석 수행
```

### 예시: 문제 해결

```
1. /gather 현재 문제 상황
   → 문제의 범위와 맥락 파악

2. /reframe
   → 문제 정의 확인

3. /truth 각 해결책의 장단점
   → 옵션별 분석
```

---

## 핵심 포인트

1. **커맨드 = Claude의 동작 모드 지정**
   - 같은 질문도 커맨드에 따라 다른 방식으로 처리

2. **순서가 있다**
   - gather(수집) → reframe(확인) → truth(분석)

3. **조합해서 사용**
   - 한 작업에 여러 커맨드 활용 가능

4. **일관된 출력**
   - 매번 같은 구조로 결과 제공

## 언제 사용하나?

| 상황 | 추천 커맨드 |
|------|-------------|
| 새 프로젝트 시작 | `/gather` |
| 요구사항 정리 | `/gather` → `/reframe` |
| 긴 대화 후 정리 | `/reframe` |
| 의사결정 필요 | `/truth` |
| 객관적 분석 필요 | `/truth` |
| 복잡한 문제 해결 | `/gather` → `/reframe` → `/truth` |
